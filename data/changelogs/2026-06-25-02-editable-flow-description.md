Admins can now add and edit a description on each approval workflow, making it easier to document and differentiate workflows across teams.

- **Editable description field:** A new text area (up to 500 characters) is available both when creating a new workflow and when editing an existing one in the Flow Details page
- **Persistent and visible:** Descriptions are saved to the backend and loaded when revisiting the configuration, so context about each workflow is always available
